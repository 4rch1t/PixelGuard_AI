6. NEURAL NETWORK ARCHITECTURE
================================================================================

MODEL CLASS: ResNet6Ch

Inheritance: nn.Module (PyTorch standard)

Base Architecture: ResNet18 pretrained on ImageNet

Modifications:
  Input channels: 3 (standard RGB) → 6 (RGB + ELA)
  Output shape: (batch, 2) → (batch, 1) for binary classification
  Output activation: None (raw logits for BCEWithLogitsLoss)

LAYER BREAKDOWN:

Layer          | Input Shape      | Output Shape     | Parameters | Purpose
conv1          | (B,6,224,224)    | (B,64,112,112)   | 9,408      | Initial feature extraction
bn1            | (B,64,112,112)   | (B,64,112,112)   | 256        | Batch normalization
maxpool        | (B,64,112,112)   | (B,64,56,56)     | 0          | Spatial reduction
layer1         | (B,64,56,56)     | (B,64,56,56)     | 148,480    | ResNet block (2 blocks)
layer2         | (B,64,56,56)     | (B,128,28,28)    | 1,219,584  | ResNet block (2 blocks) + stride
layer3         | (B,128,28,28)    | (B,256,14,14)    | 7,098,368  | ResNet block (2 blocks) + stride
layer4         | (B,256,14,14)    | (B,512,7,7)      | 14,964,736 | ResNet block (2 blocks) + stride
avgpool        | (B,512,7,7)      | (B,512,1,1)      | 0          | Global average pooling
fc             | (B,512)          | (B,1)            | 513        | Binary classification head

Total Parameters: ~11.2M (standard ResNet18: 11.2M)
Model Size: 42.75 MB (as saved checkpoint)

RESIDUAL CONNECTIONS (Skip Connections):
- Each ResNet block: input → conv → bn → relu → conv → bn + (input) → relu
- Benefit: Easier gradient flow, captures identity mappings
- Solves vanishing gradient problem

WEIGHT INITIALIZATION STRATEGY:
```python
# Load pretrained ResNet18 weights
pretrained_model = models.resnet18(weights=models.ResNet18_Weights.IMAGENET1K_V1)

# Modify first conv to accept 6 channels
old_conv = pretrained_model.conv1  # 3 input channels
new_conv = nn.Conv2d(6, 64, kernel_size=7, stride=2, padding=3)

# Copy weights: RGB channels from pretrained, duplicate for ELA
new_conv.weight[:, :3, :, :] = old_conv.weight
new_conv.weight[:, 3:, :, :] = old_conv.weight  # Duplicate ImageNet knowledge

pretrained_model.conv1 = new_conv
```

WHY DUPLICATION:
- Allows network to learn ELA-specific features from ImageNet foundation
- Better initialization than random weights
- Transfers learned texture/edge detectors to forensic domain

7. IMAGE PREPROCESSING PIPELINE
================================================================================

PREPROCESSING FUNCTION: preprocess_image(image)

INPUT: PIL Image object
OUTPUT: Torch tensor (1, 6, 224, 224)

STEP 1: ELA COMPUTATION
```python
def compute_ela_rgb(img_rgb, quality=90, enhance=15):
    # Save at Q90
    buf = io.BytesIO()
    img_rgb.save(buf, format="JPEG", quality=quality)
    buf.seek(0)
    
    # Reload and compute difference
    jpeg = Image.open(buf).convert("RGB")
    diff = ImageChops.difference(img_rgb, jpeg)
    diff = ImageEnhance.Brightness(diff).enhance(enhance)
    
    return diff
```

Mathematically:
  ELA = enhance_factor * |Original_image - Recompressed_at_Q90|

Why Q90:
  - High quality preserves real image well
  - But detects artifacts if image was previously compressed
  - 15x brightness enhancement makes differences visible

STEP 2: RESIZE TO 224x224
- Bilinear interpolation (default PIL)
- ResNet18 expects 224x224 input
- Preserves aspect ratio (with letterboxing or cropping possible)

STEP 3: CONVERT TO TENSORS
```python
rgb_tf = transforms.Compose([
    transforms.Resize((224, 224)),
    transforms.ToTensor(),  # Converts to [0,1] range (divides by 255)
])
```

STEP 4: NORMALIZE SEPARATELY
RGB normalization (ImageNet statistics):
  - mean = [0.485, 0.456, 0.406]
  - std = [0.229, 0.224, 0.225]
  - Normalization: (x - mean) / std

ELA normalization (custom):
  - mean = [0.5, 0.5, 0.5]
  - std = [0.5, 0.5, 0.5]
  - Normalization: (x - 0.5) / 0.5

Why separate normalization:
  - RGB learned from ImageNet (different distribution)
  - ELA is forensic signal (centered around 0.5)
  - Each channel type has different statistics

STEP 5: CONCATENATE CHANNELS
```python
x = torch.cat([rgb_t, ela_t], dim=0)  # (6, 224, 224)
x = x.unsqueeze(0)                    # Add batch: (1, 6, 224, 224)
```

COMPLETE PREPROCESSING PSEUDOCODE:
```
Input: image (PIL Image)
1. Convert to RGB
2. Compute ELA (recompress at Q90, enhance 15x)
3. Resize both RGB and ELA to 224x224
4. Convert to tensors [0, 1]
5. Normalize RGB with ImageNet stats
6. Normalize ELA with [0.5, 0.5, 0.5]
7. Concatenate into 6-channel tensor
8. Add batch dimension
Output: tensor (1, 6, 224, 224)
```


================================================================================
8. TRAINING PIPELINE
================================================================================

TRAINING SCRIPT: train_real_vs_ai.py

DATA LOADING:
```python
train_dataset = ImageFolder('train/', transform=train_transforms)
val_dataset = ImageFolder('val/', transform=val_transforms)

train_loader = DataLoader(train_dataset, batch_size=32, shuffle=True)
val_loader = DataLoader(val_dataset, batch_size=32)
```

LOSS FUNCTION: BCEWithLogitsLoss
```python
criterion = nn.BCEWithLogitsLoss(pos_weight=torch.tensor([1.5]))
```

Formula:
  Loss = -[y * log(sigmoid(z)) + (1-y) * log(1 - sigmoid(z))]

where:
  - z = model output (logit)
  - y = ground truth (0 for real, 1 for fake)
  - pos_weight = 1.5 (weight for positive class)

Effect of pos_weight=1.5:
  - Makes misclassifying "fake" 1.5x more costly
  - Helps with class imbalance (90% real, 10% fake)
  - Without it: Model would ignore minority class

OPTIMIZER: Adam
```python
optimizer = torch.optim.Adam(
    model.parameters(),
    lr=1e-3,           # Learning rate
    weight_decay=1e-2  # L2 regularization
)
```

Adam update rule (simplified):
  m_t = β1 * m_{t-1} + (1-β1) * g_t        (momentum)
  v_t = β2 * v_{t-1} + (1-β2) * g_t²      (second moment)
  θ_t = θ_{t-1} - lr * m_t / (√v_t + ε)   (parameter update)

LEARNING RATE SCHEDULER: OneCycleLR
```python
scheduler = torch.optim.lr_scheduler.OneCycleLR(
    optimizer,
    max_lr=1e-2,
    steps_per_epoch=len(train_loader),
    epochs=3,
    cycle_momentum=True
)
```

Learning rate schedule (3 epochs):
  Epoch 0: 1e-4 → 1e-2 (warm-up)
  Epoch 1: 1e-2 (peak learning)
  Epoch 2: 1e-2 → 1e-5 (cool-down)

Benefits:
  - Starts small (avoid divergence)
  - Peaks to explore loss landscape
  - Ends tiny (converge to minima)
  - Better generalization than constant LR

TRAINING LOOP PSEUDOCODE:
```
for epoch in range(3):
    for batch in train_loader:
        images, labels = batch
        
        # Forward pass
        outputs = model(images)
        loss = criterion(outputs, labels)
        
        # Backward pass
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
        scheduler.step()
    
    # Validation
    val_loss, val_acc = evaluate(model, val_loader)
    
    # Save best model
    if val_acc > best_acc:
        torch.save(model.state_dict(), 'best_model.pt')
```

FINAL METRICS:
- Training time: ~2 hours (3 epochs)
- Final accuracy: 93.8%
- Parameters trained: 11.2M
- Checkpoint saved: runs/smoke/best_model.pt (42.75 MB)

10. GRADCAM IMPLEMENTATION
================================================================================

CLASS: GradCAM (Explainability Module)

PURPOSE: Visualize which image regions influenced model's prediction

ALGORITHM:

1. FORWARD HOOK (capture activations):
```python
def fwd_hook(module, input, output):
    self._acts = output  # (B, C, H, W)
```

2. BACKWARD HOOK (capture gradients):
```python
def bwd_hook(module, grad_input, grad_output):
    self._grads = grad_output[0]  # (B, C, H, W)
```

3. FORWARD PASS:
```python
logit = model(x)  # Forward through entire network
```

4. BACKWARD PASS:
```python
score = logit.squeeze()
score.backward(retain_graph=False)
```

5. WEIGHT COMPUTATION:
```python
# Average gradients across spatial dimensions
weights = self._grads.mean(dim=(2, 3), keepdim=True)  # (B, C, 1, 1)
```

6. ACTIVATION WEIGHTING:
```python
# Multiply activations by weights
cam = (weights * self._acts).sum(dim=1, keepdim=True)  # (B, 1, H, W)
```

7. NORMALIZATION:
```python
cam = F.relu(cam)  # Only positive contributions
cam = cam - cam.min()
cam = cam / (cam.max() + 1e-6)  # Normalize to [0, 1]
```

OUTPUT: Heatmap (B, 1, H, W) where values in [0, 1]
- 1 (bright red): High suspicion (influenced negative prediction)
- 0 (dark blue): Low suspicion (normal features)

VISUALIZATION:
- Apply colormap (hot, jet, etc.)
- Overlay on original image
- Show to user for explainability

11. COMPRESSION ARTIFACT ANALYSIS
================================================================================

ALGORITHM: analyze_compression_artifacts(image)

STEPS:

1. CONVERT TO RGB:
```python
img_rgb = image.convert('RGB')
img_array = np.array(img_rgb, dtype=np.float32)
```

2. RECOMPRESS AT MULTIPLE QUALITIES:
```python
quality_levels = [95, 85, 75, 50]

for quality in quality_levels:
    # Recompress
    buf = io.BytesIO()
    img_rgb.save(buf, format='JPEG', quality=quality)
    buf.seek(0)
    recompressed = Image.open(buf).convert('RGB')
    recomp_array = np.array(recompressed, dtype=np.float32)
    
    # Compute MSE
    diff = np.abs(img_array - recomp_array).mean()
    quality_artifacts.append({
        'quality': quality,
        'artifact_level': diff
    })
```

3. INTERPRET PATTERN:
- REAL image: Low artifact at high Q (Q95, Q85), increases at low Q (Q50)
- FAKE image: May have irregular pattern or unexpected artifacts
- Edited image: Might show local sharp changes (edit boundary)

4. VISUALIZE:
- Plot artifact_level vs quality as line chart
- Show re-compressed versions side-by-side
- Allow user to inspect visual differences

OUTPUT:

12. METADATA EXTRACTION
================================================================================

FUNCTION: extract_metadata(image, filename)

EXTRACTED FIELDS:

1. FILE INFORMATION:
   - filename: Original file name
   - format: Image format (JPEG, PNG, etc.)
   - mode: Color mode (RGB, RGBA, L, etc.)
   - size: Dimensions (width x height)

2. EXIF DATA (via PIL.ExifTags):
   - DateTime: Capture date/time
   - Make: Camera manufacturer
   - Model: Camera model
   - Software: Software used to create/edit
   - GPS: Latitude, longitude, altitude
   - FocalLength: Zoom distance
   - Flash: Flash used (True/False)
   - And ~200 other EXIF fields

FORENSIC SIGNIFICANCE:

REAL PHOTO indicators:
✓ Valid EXIF data present
✓ Camera make/model present
✓ DateTime field
✓ GPS coordinates (optional)
✓ Logical EXIF values

FAKE IMAGE indicators:
✗ No EXIF data
✗ Generic software ("Microsoft Office", "Photoshop", etc.)
✗ Suspiciously added EXIF
✗ Impossible EXIF values
✗ Multiple conflicting EXIF entries

IMPLEMENTATION:
```python
try:
    exif_data = image._getexif()
    if exif_data:
        from PIL.ExifTags import TAGS
        for tag_id, value in exif_data.items():
            tag_name = TAGS.get(tag_id, tag_id)
            if tag_name in ['DateTime', 'Make', 'Model', 'Software']:
                metadata[tag_name.lower()] = str(value)[:100]
except:
    metadata['exif_data'] = 'Could not extract'
```

LIMITATIONS:
- EXIF easily forged by editing software
- Cannot be sole proof of authenticity
- Complements ML model, not replaces it
- Some cameras strip EXIF
- Different apps handle EXIF differently

INFERENCE TIME BREAKDOWN:

GPU (NVIDIA):
  - Preprocessing: 50 ms
  - Model forward: 1500-2000 ms (depends on GPU)
  - Postprocessing (ELA, GradCAM, forensics): 300-500 ms
  - Total: 2-2.5 seconds

CPU:

  - Model forward: 7000-8000 ms
  - Postprocessing: 300-500 ms
  - Total: 8-10 seconds

Network (Flask):
  - Request parsing: 50 ms
  - Response serialization: 200-300 ms
  - Network latency: 100-500 ms
  Total web request: 3-12 seconds (8-10s on CPU)

MEMORY USAGE:
- Model loaded: 42.75 MB
- Tensors in memory: ~500 MB (batch of 1)
- Images, cached results: ~100 MB
- Total: ~650 MB typical

OPTIMIZATION TECHNIQUES (Not yet implemented):

1. Quantization (Model Compression):
   - float32 → int8: Reduce size by 75-80%
   - Speed up by 2-3x
   - Accuracy loss: <1%

2. Pruning:
   - Remove 30-50% of non-essential weights
   - Reduce inference by 20-40%

3. Batch Processing:
   - Process multiple images simultaneously
   - Better GPU utilization
   - Reduce per-image overhead

4. Caching:
   - Cache model on disk/GPU
   - Load once, use repeatedly
   - Current: Already done (load_model() at startup)

5. TensorRT Optimization:
   - NVIDIA framework for optimized inference
   - 2-3x speedup on NVIDIA GPUs
   - Requires recompilation

6. ONNX Export:
   - Convert PyTorch → ONNX format
   - Compatible with multiple inference engines
   - Potentially faster cross-platform

CURRENT BOTTLENECK:
- ELA computation (50ms, done in Python/PIL)
- Model forward pass (1500-8000ms, depends on GPU)
- GradCAM visualization (200ms)

PRIORITIZED OPTIMIZATION:
1. Quantize model (biggest speed gain)
2. Batch processing (concurrent inference)
3. ONNX export (platform compatibility)

21. COMMON ISSUES & TROUBLESHOOTING
================================================================================

ISSUE 1: "Model not found"
Cause: runs/smoke/best_model.pt doesn't exist
Solution:
  - Check directory structure
  - Run training script: python train_real_vs_ai.py
  - Or download model checkpoint

ISSUE 2: "CUDA out of memory"
Cause: GPU doesn't have enough VRAM
Solution:
  - Reduce batch size
  - Use CPU instead: device = torch.device('cpu')
  - Reduce image size (224x224 already minimal)
  - Upgrade GPU or use cloud GPU

ISSUE 3: "OpenCV import error"
Cause: OpenCV not installed or incompatible
Solution:
  - Reinstall: pip install opencv-python==4.8.0.76
  - Check for conflicts: pip show opencv-python

ISSUE 4: "Slow inference on CPU"
Expected: 8-10 seconds normal
Consider:
  - Use GPU instead
  - Deploy to cloud with GPU
  - Quantize model for 2-3x speedup

ISSUE 5: "PDF generation fails"
Cause: ReportLab image encoding issue
Solution:
  - Check image format (PNG required)
  - Verify image not corrupted
  - Check disk space for temp files

ISSUE 6: "Flask endpoint not responding"
Cause: Model still loading, stuck inference
Solution:
  - Check GPU memory
  - Verify proper CUDA installation
  - Look at server logs for errors
